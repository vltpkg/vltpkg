Open documentation for a package in a web browser.

## How It Works

The `vlt docs` command intelligently locates and opens the
documentation for any package in your dependency graph. It follows a
smart resolution strategy to find the best documentation URL:

1. **Repository Resolution** – Checks the package's `repository` field
   in its `package.json`
2. **Git Host Detection** – For known git hosts (GitHub, GitLab,
   Bitbucket, etc.), generates the canonical documentation URL
3. **Fallback** – If no repository is found or the git host is
   unknown, falls back to the npm package page

This means you always get taken to the most relevant documentation,
whether it's the project's README on GitHub, their custom docs site,
or the npm registry page.

## Basic Usage

### Open docs for current project

When run without arguments, `vlt docs` reads your local `package.json`
and opens the documentation for your current project:

<Code code="$ vlt docs" title="Terminal" lang="bash" />

This is particularly useful when working on a project and you want to
quickly view your own documentation or verify your repository URL is
correctly configured.

### Open docs for any package

Provide a package spec to open documentation for any package, whether
it's installed in your project or not:

<Code
  code={`# Open docs for a specific package
$ vlt docs express

# Open docs for a specific version

$ vlt docs typescript@5.0.0

# Open docs for a scoped package

$ vlt docs @vltpkg/graph`} title="Terminal" lang="bash" />

The command fetches the package manifest from the registry and
intelligently determines the best documentation URL to open.

## Repository URL Resolution

The command prioritizes finding documentation in this order:

1. **Git Repository README** – For packages hosted on GitHub, GitLab,
   Bitbucket, and other known git hosts, opens the repository's
   documentation page (typically the README)
2. **npm Package Page** – For packages without a recognized repository
   or with unknown git hosts, opens the package page on npmjs.com

### Git Host Support

The following git hosts are automatically recognized and converted to
their documentation URLs:

- GitHub (`github.com`)
- GitLab (`gitlab.com`)
- Bitbucket (`bitbucket.org`)
- SourceForge (`sourceforge.net`)
- Gitea instances
- And others supported by the `hosted-git-info` library

## Notes

- The command automatically opens your default web browser
- Works with any valid package specifier (names, versions, tags, etc.)
- Does not require the package to be installed in your project
- Falls back gracefully to npm package pages when repository info is
  unavailable
