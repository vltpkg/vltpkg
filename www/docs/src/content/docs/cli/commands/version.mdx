Bump a package's version.

Run in a package directory to bump the version and write the new data
back to package.json.

The `<newversion>` argument should be a valid semver string or a valid
increment type (one of patch, minor, major, prepatch, preminor,
premajor, prerelease).

If run in a git repository, it will also create a version commit and
tag.

## Examples

Increment the patch version:

<Code code="$ vlt version patch" title="Terminal" lang="bash" />

Increment the minor version:

<Code code="$ vlt version minor" title="Terminal" lang="bash" />

Increment the major version:

<Code code="$ vlt version major" title="Terminal" lang="bash" />

Increment the prerelease version:

<Code code="$ vlt version prerelease" title="Terminal" lang="bash" />

Set the version to a specific semver string:

<Code code="$ vlt version 1.2.3" title="Terminal" lang="bash" />

Increment to a pre-major version:

<Code code="$ vlt version premajor" title="Terminal" lang="bash" />

Increment to a pre-minor version:

<Code code="$ vlt version preminor" title="Terminal" lang="bash" />

Increment to a pre-patch version:

<Code code="$ vlt version prepatch" title="Terminal" lang="bash" />
