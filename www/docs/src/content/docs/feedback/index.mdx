## Report an issue

<Steps>
  <ol>
    <li>
      Check whether the issue already exists by searching our [GitHub
      issue tracker](https://github.com/vltpkg/vltpkg/issues). If you
      find a matching report, add a 👍 or a comment with your details
      instead of opening a duplicate.
    </li>
    <li>
      If nothing matches, open a new issue as a [feature
      request](https://github.com/vltpkg/vltpkg/issues/new?template=feature_request.yml)
      or a [bug
      report](https://github.com/vltpkg/vltpkg/issues/new?template=bug_report.yml).
    </li>
  </ol>
</Steps>

If the `vlt` CLI runs into an unexpected error, it writes the full
details to an `error-<pid>.log` file and points you to the same
tracker — attach that log to your bug report to help us reproduce it.
