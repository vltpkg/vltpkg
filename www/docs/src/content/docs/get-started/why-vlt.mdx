## The ways infrastructure that underlies open source have simply stopped working

Since late 2025, the Node ecosystem and critical open source
maintainers have been targets of an unprecedented number of cyber
attacks.[1] These attacks breached over 25,000 GitHub repositories,
and exposed over 40,000 sensitive credentials across AWS, Azure, and
GCP cloud environments. (We won't name them, but you can imagine many
an Important SaaS Vendor was impacted ;) )

Despite 63% download growth,
[Sonatype's State of Supply Chain 2026](https://www.sonatype.com/state-of-the-software-supply-chain/introduction)
found 838,778 packages published with a CVSS 9.0+ score. AI assisted
development can only compound more when human review and the
faultlines of existing platforms can no longer keep up with the
demands for release, recovery and verification where it matters.

> We're doing this because we expect more from systems we've always
> counted on to host and distribute our software.

## Move fast and stay safe

vlt is a toolset created by seasoned developers of the core npm team,
who have spent decades of their work lives enduring maintenance
strife.

- First class support for Javascript and Typescript projects: Whether
  you use Next, Vue, Svelte... we run our benchmarks against projects
  for those frameworks.

- Blazingly fast per-package installations: 1.1ms for installing
  `node_modules`! vlt drops CI compute time and bandwidth at scale.

- Slimmer packuments: the vlt registry serves 76% smaller packuments
  than han the ten thousand most downloaded npm packages.

## Safer installations

Our CLI separates run and build steps through a two-phase install
(first, download, then opt-in build), which prevents potential
malicious lifecycle lifecycle scripts from running by default.

- Integrity caching for performance: we use content-addressable
  storage which reverifies a package per fetch request.

All of this works with your favorite npm-compatible package managers!
No extra tooling or migration needed.
