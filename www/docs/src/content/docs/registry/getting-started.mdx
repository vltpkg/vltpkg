### CLI Options

| Option     | Alias | Default | Description             |
| ---------- | ----- | ------- | ----------------------- |
| `--port`   | `-p`  | `1337`  | Server port             |
| `--config` | `-c`  | -       | Config file path        |
| `--debug`  | `-d`  | `false` | Debug mode              |
| `--daemon` | -     | `true`  | Local filesystem daemon |
| `--help`   | `-h`  | -       | Show help               |

## Requirements

**Development:**

- Node.js (latest LTS)
- VLT package manager

**Production:**

- Cloudflare Account (free tier available)

## ğŸš€ Production Deployment

Deploy VSR to Cloudflare Workers with the new `vsr deploy` command:

<Code code="# Deploy to default environment (dev)
vsr deploy

# Deploy to production

vsr deploy --env=prod

# Preview deployment without actually deploying

vsr deploy --dry-run

# Override specific resource names

vsr deploy --env=staging --db-name=my-custom-db
--bucket-name=my-custom-bucket" lang="bash" />

### Deploy Options

| Option          | Default | Description                    |
| --------------- | ------- | ------------------------------ |
| `--env`         | `dev`   | Environment (dev/staging/prod) |
| `--db-name`     | -       | Override D1 database name      |
| `--bucket-name` | -       | Override R2 bucket name        |
| `--queue-name`  | -       | Override queue name            |
| `--dry-run`     | `false` | Preview deployment             |
